Conceptos básicos antes de comenzar

Esta práctica no asume conocimientos previos. La idea es entender primero qué hace cada herramienta.

¿Qué es Hadoop?

Hadoop es una herramienta para almacenar y procesar grandes cantidades de datos. En esta práctica usaremos una versión lista para aprender.

¿Qué es Docker?

Docker permite ejecutar programas dentro de contenedores. Eso evita instalar Hadoop manualmente desde cero.

¿Qué es HDFS?

HDFS es el sistema de archivos de Hadoop. Ahí se guardan los archivos que luego Hadoop puede usar.

NameNode y DataNode

El NameNode organiza la información y el DataNode guarda los datos realmente.

Objetivo de la práctica: clonar el repositorio base del curso, abrirlo en Visual Studio Code, levantar Hadoop con Docker, subir datos_200mb.csv a HDFS y comprobar el resultado por terminal y por navegador.

Requisitos previos

Antes de ejecutar comandos, instala Visual Studio Code, Git y deja Docker Desktop listo.

  • Docker Desktop instalado.
  • Visual Studio Code.
  • Git instalado.
  • Terminal disponible.
Windows
  1. Instala Visual Studio Code desde code.visualstudio.com.
  2. Instala Git desde git-scm.com/download/win.
  3. Abre VS Code y usa la terminal integrada, PowerShell o Git Bash.
  4. Abre Docker Desktop antes de iniciar la práctica.
Mac
  1. Instala Visual Studio Code desde code.visualstudio.com.
  2. Instala Git desde git-scm.com/download/mac o verifica si ya existe con git --version.
  3. Usa la terminal integrada de VS Code o la app Terminal.
  4. Abre Docker Desktop y espera a que el motor esté activo.

Paso a paso para Windows y Mac

Los comandos principales son los mismos en Windows y Mac. Lo más importante es estar en la carpeta correcta, trabajar desde Visual Studio Code y mantener Docker Desktop abierto.

Advertencia importante:
Si el comando inicia con docker, normalmente se ejecuta desde tu computadora.
Si el comando inicia con hdfs dfs, normalmente se ejecuta dentro del contenedor namenode.
1

Verificar que Docker funciona

Este es el primer control. Ejecuta estos comandos en la terminal de tu computadora, estando dentro de la carpeta hadoop_isil.

Ejecutar en la terminal de tu computadora - carpeta hadoop_isil 
docker --version
docker compose version
docker ps

Debe aparecer una versión de Docker, una versión de Docker Compose y una tabla de contenedores. Si la tabla está vacía, no es error.

2

Preparar la carpeta base del proyecto

Después de clonar tu repositorio base y abrirlo en Visual Studio Code, la carpeta principal de trabajo será hadoop_isil.

Estructura esperada 
hadoop_isil/
├── docker-hadoop/
├── datos_200mb.csv
├── README.md
├── index.html
└── comandos-hadoop.txt

Regla simple:

  • Para levantar Hadoop, entra a docker-hadoop.
  • Para copiar el archivo CSV al contenedor, debes estar en hadoop_isil o usar la ruta correcta.
3

Descargar el entorno Hadoop dentro de la carpeta base

Dentro de hadoop_isil, descarga el repositorio big-data-europe/docker-hadoop para tener el entorno Hadoop listo.

Ejecutar en la terminal de tu computadora - carpeta hadoop_isil 
git clone https://github.com/big-data-europe/docker-hadoop.git
cd docker-hadoop
4

Levantar Hadoop

Este comando inicia los contenedores en segundo plano. La primera vez puede tardar varios minutos porque Docker descargará imágenes. Para este paso debes entrar a la carpeta docker-hadoop.

Ejecutar en la terminal de tu computadora - entrar a docker-hadoop 
cd docker-hadoop
docker compose up -d
5

Confirmar que los contenedores están arriba

Busca nombres como namenode, datanode, resourcemanager, nodemanager y historyserver. Puedes ejecutar este comando desde tu computadora mientras sigues dentro de docker-hadoop.

Ejecutar en la terminal de tu computadora - carpeta docker-hadoop 
docker ps

Lo que se ve: Docker Desktop con el grupo docker-hadoop y varios contenedores activos, por ejemplo namenode, datanode, nodemanager e historyserver. Si ves los puntos en verde, el entorno ya está corriendo.

6

Confirmar que el navegador abre Hadoop

Después de levantar los contenedores, comprueba que Hadoop responde en el navegador. La dirección principal será http://localhost:9870.

Direcciones del navegador 
http://localhost:9870
http://localhost:8088
  • localhost:9870 muestra el NameNode y permite explorar HDFS.
  • localhost:8088 muestra YARN / ResourceManager.

Si 8088 tarda más en abrir, primero confirma 9870. La validación principal de HDFS se hace en 9870.

Comandos principales

Este bloque resume la práctica por contexto para que no confundas comandos de computadora con comandos de Hadoop.

Regla rápida: si estás fuera del contenedor, usa comandos docker. Si ya entraste a namenode, usa comandos hdfs dfs.
A. Ejecutar en la terminal de tu computadora - carpeta hadoop_isil 
git clone https://github.com/GxJohan/hadoop_isil.git
cd hadoop_isil
code .
docker --version
docker compose version
docker ps
git clone https://github.com/big-data-europe/docker-hadoop.git
ls -lh
docker cp datos_200mb.csv namenode:/tmp/datos_200mb.csv
docker exec -it namenode bash
B. Ejecutar en la terminal de tu computadora - carpeta docker-hadoop 
cd docker-hadoop
docker compose up -d
docker ps
docker cp ../datos_200mb.csv namenode:/tmp/datos_200mb.csv
docker exec -it namenode bash
docker compose down
C. Ejecutar dentro del contenedor namenode 
hdfs dfs -mkdir -p /data
hdfs dfs -put -f /tmp/datos_200mb.csv /data/
hdfs dfs -ls -h /data
hdfs dfs -head /data/datos_200mb.csv
exit

Prueba con archivo pesado

Esta es la práctica central. Todo el trabajo irá directamente al archivo pesado y a la ruta /data de HDFS.

El archivo fue descargado desde:
https://examplefile.com/code/csv/200-mb-csv

Importante: el nombre descargado puede variar. Para esta guía usaremos el nombre datos_200mb.csv.

Flujo de la práctica: repositorio base en VS Code -> archivo local -> contenedor Docker -> HDFS -> navegador.

Paso 1 - Verificar el nombre del archivo descargado 
# Mac / Linux
ls -lh

# Windows PowerShell
dir

Si el archivo no se llama datos_200mb.csv, puedes renombrarlo para que los comandos de la práctica coincidan exactamente con la guía.

Paso 2 - Renombrar el archivo si tiene otro nombre 
# Mac / Linux
mv nombre_original.csv datos_200mb.csv

# Windows PowerShell
ren nombre_original.csv datos_200mb.csv

Ahora decide desde qué carpeta estás trabajando:

  • Si estás en hadoop_isil, usa el archivo como datos_200mb.csv.
  • Si estás dentro de docker-hadoop, usa la ruta ../datos_200mb.csv.
Paso 3A - Ejecutar en la terminal de tu computadora - carpeta hadoop_isil 
docker cp datos_200mb.csv namenode:/tmp/datos_200mb.csv
docker exec -it namenode bash
Paso 3B - Ejecutar en la terminal de tu computadora - carpeta docker-hadoop 
docker cp ../datos_200mb.csv namenode:/tmp/datos_200mb.csv
docker exec -it namenode bash

Después de entrar con docker exec -it namenode bash, ya estás dentro del contenedor.

Paso 4 - Ejecutar dentro del contenedor namenode 
hdfs dfs -mkdir -p /data
hdfs dfs -put -f /tmp/datos_200mb.csv /data/
hdfs dfs -ls -h /data
hdfs dfs -head /data/datos_200mb.csv
Paso 5 - Salir del contenedor 
exit

Resultado esperado: después de subirlo, debes poder ver datos_200mb.csv dentro de /data, primero por consola y luego en http://localhost:9870.

Errores comunes

Estos son los problemas más habituales cuando recién se empieza.

Problema Qué hacer
Docker no abre. Abre Docker Desktop otra vez y espera a que quede operativo.
Puerto 9870 no carga. Revisa docker ps, espera un poco y confirma que namenode está corriendo.
docker compose no existe. Actualiza Docker Desktop. En versiones antiguas puede existir docker-compose.
El contenedor namenode no aparece. Asegúrate de estar dentro de la carpeta docker-hadoop y repite docker compose up -d.
El archivo no se ve en HDFS. Confirma con hdfs dfs -ls -h /data y actualiza el navegador.
En Windows el comando no funciona por estar en otra carpeta. Usa cd para entrar a la carpeta correcta. Puedes confirmar con dir o ls.
Estoy dentro del contenedor y escribo docker ps, pero no funciona. Sal con exit y ejecuta docker ps desde la computadora.
Estoy en docker-hadoop y no encuentra datos_200mb.csv. Usa ../datos_200mb.csv o vuelve a la carpeta raíz hadoop_isil.
El archivo descargado no se llama datos_200mb.csv. Verifica el nombre con dir o ls -lh y luego renómbralo.
El navegador 9870 abre, pero no veo el archivo. Confirma primero que el archivo sí se subió a HDFS con hdfs dfs -ls -h /data.
