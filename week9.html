Semana 9 - Spark SQL conectado a Hadoop

Guía práctica para reutilizar el clúster Hadoop del proyecto hadoop_isil, confirmar el dataset en HDFS y trabajar con Spark SQL leyendo ai_student_impact_dataset.csv desde HDFS.

Semana 9 Tema principal: Spark SQL Hadoop + HDFS + PySpark Dataset en HDFS

Antes de Empezar

Esta semana no empieza desde cero. Primero se deja listo HDFS y luego se conecta Spark al mismo entorno.

Idea clave: si el archivo ai_student_impact_dataset.csv ya aparece en /data dentro de HDFS, recién ahí tiene sentido entrar a PySpark.

Carpeta correcta

Trabaja desde la raíz hadoop_isil.

Archivo requerido

Debe existir ai_student_impact_dataset.csv en la raíz del proyecto.

Docker activo

Docker Desktop debe estar levantado antes de ejecutar comandos.

Clúster correcto

Debes usar el entorno de docker-compose.week7.yml, no solo docker-hadoop.

Importante: el bloque de HDFS de abajo es el punto mínimo necesario para confirmar que Spark luego podrá leer el archivo desde hdfs://namenode:9000/data/....

Bloque Base: dejar HDFS listo

Este bloque viene de la práctica anterior y se mantiene igual. Hasta aquí debes llegar antes de hablar de Spark SQL.

1

Levantar el entorno Hadoop

Hazlo desde la raíz hadoop_isil.

Terminal de tu computadora
Comandos 
docker compose -f docker-compose.week7.yml up --build -d
docker compose -f docker-compose.week7.yml ps
Debes ver: namenode, datanode, resourcemanager, nodemanager e historyserver levantados.
2

Subir el dataset a HDFS

El archivo debe llamarse ai_student_impact_dataset.csv y estar en la raíz.

Terminal de tu computadora
Comandos 
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -mkdir -p /data
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -put -f /workshop/ai_student_impact_dataset.csv /data/
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -ls -h /data
docker exec namenode sh -lc '/opt/hadoop-3.2.1/bin/hdfs dfs -head /data/ai_student_impact_dataset.csv | head -n 5'
Debes ver: el archivo dentro de /data y las primeras filas del CSV al final.
3

Confirmación mínima para seguir

Si esto sale bien, ya puedes pasar a Spark SQL.

  • El clúster está levantado.
  • El archivo aparece en /data.
  • hdfs dfs -head devuelve filas reales del CSV.

Paso a Paso de Spark SQL

A partir de aquí el objetivo ya no es MapReduce. El objetivo es leer el CSV desde HDFS con Spark y consultar con SQL.

4

Verifica que Hadoop siga levantado

Haz esta comprobación antes de abrir Spark.

Terminal de VS Code
Comando 
docker ps
Debes ver algo como: namenode, datanode, resourcemanager, nodemanager e historyserver.
5

Verifica otra vez que el archivo está en HDFS

Aquí entras a namenode y revisas desde dentro.

Terminal de tu computadora y luego contenedor
Comandos 
docker exec -it namenode bash

hdfs dfs -ls -h /data
hdfs dfs -head /data/ai_student_impact_dataset.csv
Si aparece el archivo: ya puedes conectar Spark. Luego sal con exit.
6

Descarga la imagen de Spark con Python

También conviene revisar el nombre de la red Docker donde vive Hadoop.

Terminal de tu computadora
Comandos 
docker pull apache/spark-py
docker network ls
Debes identificar: la red del proyecto. En este repositorio normalmente será hadoop_isil_default.
7

Entra a PySpark dentro de la misma red de Hadoop

Si la red tiene otro nombre, reemplázala.

Terminal de tu computadora
Comando 
docker run -it --rm \
  --name spark-sql \
  --network hadoop_isil_default \
  apache/spark-py \
  /opt/spark/bin/pyspark
Con esto entras: a una consola interactiva de PySpark.
8

Crea la sesión Spark apuntando a HDFS

Este bloque ya se ejecuta dentro de PySpark.

Dentro de PySpark
PySpark 
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("ISIL_Spark_SQL") \
    .config("spark.hadoop.fs.defaultFS", "hdfs://namenode:9000") \
    .getOrCreate()
9

Lee el CSV desde HDFS

Usa opciones seguras para evitar problemas de lectura.

Dentro de PySpark
PySpark 
df = spark.read \
    .option("header", "true") \
    .option("multiline", "true") \
    .option("escape", '"') \
    .csv("hdfs://namenode:9000/data/ai_student_impact_dataset.csv")

df.printSchema()
df.show(10, truncate=False)
Debes ver: la estructura de columnas y luego las primeras 10 filas.
10

Crea la vista temporal y empieza con SQL

Desde aquí ya trabajas como Spark SQL.

Dentro de PySpark
PySpark + SQL 
df.createOrReplaceTempView("personas")

spark.sql("SELECT * FROM personas LIMIT 10").show(truncate=False)

Cuatro Consultas Siguientes sobre la Data

Después de confirmar que Spark ya leyó el CSV, puedes seguir con consultas SQL simples sobre la vista personas.

11

Contar registros totales

Dentro de PySpark
SQL 
spark.sql("""
SELECT COUNT(*) AS total_registros
FROM personas
""").show()
12

Contar estudiantes por categoría

Dentro de PySpark
SQL 
spark.sql("""
SELECT Major_Category, COUNT(*) AS total_estudiantes
FROM personas
GROUP BY Major_Category
ORDER BY total_estudiantes DESC
""").show(truncate=False)
13

Promedio de GPA antes y después

Dentro de PySpark
SQL 
spark.sql("""
SELECT
  ROUND(AVG(CAST(Pre_Semester_GPA AS DOUBLE)), 3) AS promedio_pre_gpa,
  ROUND(AVG(CAST(Post_Semester_GPA AS DOUBLE)), 3) AS promedio_post_gpa
FROM personas
""").show(truncate=False)
14

Horas promedio de uso de GenAI por categoría

Dentro de PySpark
SQL 
spark.sql("""
SELECT
  Major_Category,
  ROUND(AVG(CAST(Weekly_GenAI_Hours AS DOUBLE)), 2) AS promedio_horas_genai
FROM personas
GROUP BY Major_Category
ORDER BY promedio_horas_genai DESC
""").show(truncate=False)
15

Top 10 estudiantes con más horas de GenAI

Dentro de PySpark
SQL 
spark.sql("""
SELECT
  Student_ID,
  Major_Category,
  Weekly_GenAI_Hours,
  Primary_Use_Case
FROM personas
ORDER BY CAST(Weekly_GenAI_Hours AS DOUBLE) DESC
LIMIT 10
""").show(truncate=False)

Errores Comunes

Antes de asumir que Spark está mal, revisa primero estas causas frecuentes.

No aparece el archivo en HDFS

No sigas con Spark. Primero vuelve al bloque de carga en /data.

La red Docker no coincide

Revisa docker network ls y usa el nombre real en docker run.

Spark no encuentra namenode

Casi siempre significa que el contenedor de Spark no entró a la misma red que Hadoop.

El CSV carga raro

Usa las opciones header, multiline y escape tal como están en esta guía.

Idea Principal de Semana 9

Primero confirmas que HDFS ya tiene el dataset. Luego conectas Spark a ese mismo HDFS y consultas el CSV con SQL.

Resumen: Hadoop sigue siendo el almacenamiento distribuido y Spark SQL se convierte en la capa de consulta sobre ai_student_impact_dataset.csv.
