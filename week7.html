Paso a Paso

Sigue el orden. Cada bloque indica qué debes ejecutar y qué deberías observar.

1

Levantar el entorno de Semana 7

Hazlo desde la carpeta raíz hadoop_isil.

Ejecutar en la terminal de tu computadora
Comandos 
docker compose -f docker-compose.week7.yml up --build -d
docker compose -f docker-compose.week7.yml ps
Debes ver: contenedores como namenode, datanode, resourcemanager, nodemanager e historyserver en estado saludable.
2

Confirmar que Python está dentro del clúster

No se usa el Python local. Python debe estar dentro de Docker.

Ejecutar en la terminal de tu computadora
Comandos 
docker exec namenode sh -lc 'python3 --version && python --version'
docker exec nodemanager sh -lc 'python3 --version && python --version'
Debes ver: una versión de Python en ambos contenedores. En esta práctica validada es Python 3.5.3.
3

Subir el dataset a HDFS

El archivo debe llamarse ai_student_impact_dataset.csv y estar en la raíz.

Ejecutar en la terminal de tu computadora
Comandos 
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -mkdir -p /data
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -put -f /workshop/ai_student_impact_dataset.csv /data/
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -ls -h /data
docker exec namenode sh -lc '/opt/hadoop-3.2.1/bin/hdfs dfs -head /data/ai_student_impact_dataset.csv | head -n 5'
Debes ver: el archivo dentro de /data y las primeras filas del CSV al final.
4

Probar el mapper y el reducer con una muestra corta

Esta prueba rápida ayuda a detectar errores antes del job completo.

Ejecutar en la terminal de tu computadora
Comando 
docker exec namenode sh -lc 'head -n 20 /workshop/ai_student_impact_dataset.csv | python3 /workshop/week7/scripts/mapper_major_category.py | sort | python3 /workshop/week7/scripts/reducer_count.py'
Debes ver: categorías como Arts, Business, Humanities, Medical y STEM con números pequeños.
5

Ejecutar el trabajo real de Hadoop Streaming

Este es el MapReduce completo. Primero limpia la salida anterior y luego lanza el job.

Ejecutar en la terminal de tu computadora
Limpiar salida anterior 
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -rm -r -f /output_major_category
Ejecutar el job 
docker exec namenode sh -lc '/opt/hadoop-3.2.1/bin/hadoop jar /opt/hadoop-3.2.1/share/hadoop/tools/lib/hadoop-streaming-3.2.1.jar -files /workshop/week7/scripts/mapper_major_category.py,/workshop/week7/scripts/reducer_count.py -mapper "python3 mapper_major_category.py" -reducer "python3 reducer_count.py" -input /data/ai_student_impact_dataset.csv -output /output_major_category'
Debes ver: mensajes de avance del job hasta llegar a map 100% reduce 100%.
6

Revisar el resultado final

Aquí compruebas que Hadoop procesó el dataset y dejó una salida en HDFS.

Ejecutar en la terminal de tu computadora
Comandos 
docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -ls -h /output_major_category
docker exec namenode sh -lc '/opt/hadoop-3.2.1/bin/hdfs dfs -cat /output_major_category/part-00000 | head -n 20'
Resultado validado en esta implementación: 
Arts       5933
Business   12538

Qué Significa el Resultado

El trabajo cuenta cuántos estudiantes pertenecen a cada categoría de carrera.

Mapper

Lee cada fila del CSV y extrae el valor de Major_Category.

Shuffle de Hadoop

Agrupa automáticamente las categorías iguales antes de enviarlas al reducer.

Reducer

Suma cuántas veces apareció cada categoría y guarda el resultado final en HDFS.

Salida

El archivo final queda en /output_major_category/part-00000.

  • Revisar /output_major_category para ver part-00000
D

Si no aparece la salida

Vuelve a comprobar por consola para confirmar si el job terminó bien.

Verificación rápida

Errores Comunes

Si algo no sale como esperas, revisa esta tabla antes de repetir la práctica.

Problema Qué significa Qué hacer
python3 --version falla en el contenedor El stack de Semana 7 no se levantó con las imágenes correctas. Vuelve a ejecutar docker compose -f docker-compose.week7.yml up --build -d.
No encuentra ai_student_impact_dataset.csv El archivo no está en la raíz hadoop_isil o tiene otro nombre. Mueve o renombra el archivo para que quede como hadoop_isil/ai_student_impact_dataset.csv.
El job no crea /output_major_category Puede haber fallado el job o ya existía una salida previa. Ejecuta primero docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -rm -r -f /output_major_category y luego vuelve a lanzar el job.
localhost:9870 abre, pero no ves la salida Puede que el archivo sí exista, pero aún no estés en la ruta correcta. Busca /output_major_category en HDFS o verifica por consola con docker exec namenode /opt/hadoop-3.2.1/bin/hdfs dfs -ls -h /output_major_category.
En Windows o macOS el comando parece largo Es normal. El flujo usa comandos completos desde la computadora. Usa los botones de copiar y pega cada bloque completo en tu terminal.

Idea Principal de Esta Práctica

Hadoop sigue ejecutándose dentro de Docker, HDFS guarda el archivo y Hadoop Streaming usa Python para el procesamiento distribuido.

Resumen: en Semana 7 no empiezas de cero. Actualizas hadoop_isil, subes un dataset limpio a HDFS y ejecutas un MapReduce real con Python.
